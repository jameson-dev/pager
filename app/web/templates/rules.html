Parsing Rules

Rules run top-to-bottom; the first applicable match wins. Use Keywords to pick which message format a rule is for — the rule only applies when the message contains one of its keywords (comma-separated; leave blank to consider the rule for every message). Then write a normal regex and wrap the parts you want to capture in plain ( ) groups, and assign each group to a field. You don't need the (?P<name>...) syntax; it's added for you. Edits are tested live; nothing is applied until you Save rules.
