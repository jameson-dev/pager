Printing

Control whether and where jobs print, and which capcodes and job types are allowed to print.

General

Master switch. A job prints only if this is on and its capcode is enabled and its job type is enabled.

Monitored Capcodes

Only pages from these capcodes are processed. The alias is a friendly name shown in notifications, the Jobs list, and on the PDF (as the capcode_alias field). Per-capcode print toggle and copy count combine with the global switch.

Mute printing for specific job types (matched from the jobtype field). Types not listed default to printing enabled.

Templates

The PDF that fields are stamped onto. Upload one or more and choose the active template.

NamePDF

Uploaded PDFs are stored on the server under <config>/templates/. Remember to Save settings after uploading or selecting the active template.

Alerts & Reliability

New-job notifications, the feed watchdog, and automatic cleanup of old jobs.

Alerts & Watchdog

If set, a JSON POST is sent when a print fails permanently or the feed goes stale — so problems are noticed without a browser tab open.

Retention

Automatically delete old jobs. Runs daily and on startup.

System

File locations used by the watcher and output.

Ingest source

Where pages come from. Log file tails multimon-ng's output; PagerMon database reads PagerMon's own SQLite messages.db directly (read-only) and adapts to its schema.

Manual column mapping (only if auto-detect is wrong)

Paths & Time

Sets the timezone for the date/time/datetime fields stamped on PDFs and shown in alerts.

Print retry

How a failed print is retried by the background worker before it's given up on (and an alert is sent).

Access Password

A single shared password blocks all access to the site until entered.
